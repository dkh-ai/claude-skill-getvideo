# Guardrails для самоулучшения

## Правила, которые НЕЛЬЗЯ менять автоматически
1. Никогда не коммить напрямую в main/master
2. Никогда не удалять тесты (только добавлять)
3. Не менять CI/CD конфигурацию без ревью человека
4. Не обновлять зависимости мажорных версий автоматически
5. Не менять .env, секреты, credentials

## Лимиты на автоматические изменения
- Максимум 10 файлов за одну итерацию
- Максимум 500 строк изменений за итерацию
- Каждое изменение должно проходить тесты
- Откат если тесты падают после изменения

## Обязательный human review для:
- Изменения в CLAUDE.md секции "Критические правила"
- Новые зависимости
- Изменения в конфигурации безопасности
- Удаление любых файлов
- Изменения в схемах баз данных

## Версионирование
- CLAUDE.md хранится в git с полной историей
- Каждое автоматическое изменение = отдельный коммит с пояснением
- Формат коммита: `[self-improve] описание изменения`
